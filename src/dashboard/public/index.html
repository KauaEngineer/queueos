QUEUEOS

Async Queue & Job Processing System

Offload heavy tasks (emails, PDFs, image processing, notifications) from your main app to dedicated worker pools. Built on top of Node.js, Redis and BullMQ, with a real-time dashboard, multi-tenancy, cron scheduling, Prometheus metrics and Kubernetes manifests ready to deploy.

03 · Features

What's inside

⚡ 4 specialized queues

email-sender, report-gen, image-proc and notifications — each with their own concurrency, timeout and retry policy.

📊 Real-time dashboard

Live cards, throughput chart, queue states, recent jobs and worker stats updating every 2 seconds via Redis polling.

⏰ Cron scheduling

Schedule recurring jobs with standard cron patterns straight from the UI. Powered by BullMQ repeat.

👥 Multi-tenant

Isolate jobs and metrics per tenant via the X-Tenant-Id header. Filter the dashboard live.

🚨 Webhook alerts

POST to an external URL (Slack, Discord, Telegram) whenever 3+ jobs fail within 60 seconds.

📈 Prometheus + Grafana

/metrics endpoint in Prometheus exposition format. K8s manifests with HPA included.

05 · Architecture

How it works

[API / App]  ──enqueue──▶  [Redis Broker]  ──consume──▶  [Workers x4 pools]
     │                          │                              │
 (HTTP req)            (Sorted Sets / Lists)              (Process Jobs)
     │                          │                              │
[BullMQ Queue Mgr]                                  [Prisma → PostgreSQL]
                                                              │
                              [Dashboard]  ◀──poll──  [Redis Stats + DB]

06 · Tech Stack

Built with battle-tested tools

Node.js 20 LTS

Non-blocking runtime, ideal for I/O-heavy job processing.

Redis 7

In-memory message broker storing queues as sorted sets and lists.

BullMQ 5

Queue library with retries, delayed jobs, priority and rate limiting.

TypeScript 6

Static typing with strongly-typed job interfaces.

Prisma 6

Type-safe ORM managing migrations and DB access.

PostgreSQL 16

Relational store for job logs, queue snapshots and audit history.
